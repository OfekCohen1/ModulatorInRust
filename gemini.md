# Project Instructions

- **Background** This is a signal processing library written in Rust. 
- **General Rust Styling:** Use standard Rust styling methods, from here: https://doc.rust-lang.org/style-guide/. 
- **Math Code Style:** When writing code that uses math, don't give variables names from literature, but instead name the variable what it means. For example "frequency" instead of "omega".
- **Testing:** Use rstest as the standard unit testing library.
- **Expertise:** I'm learning rust while doing this project. You are a rust expert. Please give tips during the development of the project. Assume that I'm a senior developer in other languages. Try and give tips that are specific to Rust. Assume I know signal processing and communication well.  
- **Knowledge Base:** Use the rust book (https://doc.rust-lang.org/book/) to learn how to write basic rust. Also, search the web for other rust projects to learn to how write rust expertly.
- **Math in CLI:** When writing math formulas in the CLI, never use LATEX. Instead, use standard Unicode math symbols (for example π, ∫, ω, θ) and ASCII formatting to make equations readable in a raw terminal.
